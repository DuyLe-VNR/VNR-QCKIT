VNR Development Pipeline
DevKit → QCKit

Trang tổng quan nối hai tài liệu pipeline hiện có: DevKit chịu trách nhiệm biến yêu cầu thành thiết kế, mã nguồn và báo cáo kỹ thuật; QCKit tiếp nhận artifacts để khám phá UI, mapping Page Object, sinh test, chạy automation và triage kết quả.

DevKit trước QCKit sau Artifacts làm điểm bàn giao HITL tại các cổng chất lượng
Mở DevKit Overview Mở QCKit Overview

1. Luồng Development Pipeline tổng thể

DevKit tạo nền tảng triển khai và artifacts kỹ thuật. Sau khi mã nguồn, testcase và report sẵn sàng, QCKit tiếp tục ở lớp kiểm thử tự động để xác minh hành vi thực tế của ứng dụng.

Phase 1

DevKit - Build & Review

Orchestrate từ yêu cầu feature đến plan, tasks, testcase, triển khai, review và báo cáo.

1

Plan & Tasks

Tạo plan.md, data-model.md, contracts và tasks.md.

2

Implement

Triển khai backend, frontend, kiểm tra build và cập nhật mã nguồn.

3

Review & Report

Review code, kiến trúc, bảo mật, E2E stub và xuất final report.

→
Phase 2

QCKit - Test & Triage

Tiếp nhận feature scope và artifacts để tự động hóa kiểm thử trên UI thật bằng Playwright.

4

Discover & Map

Crawl URL, DOM, component, user flow và sinh Page Object.

5

Generate & Execute

Sinh test case, dữ liệu test, spec Playwright, trace và screenshot.

6

Triage

Phân loại PASS, APP_BUG, INFRA và quyết định hành động tiếp theo.

Điểm bàn giao chính: specs/<feature>/plan.md, tasks.md, testcases.md, mã nguồn đã build, E2E stub và final-report.md. QCKit dùng các đầu ra này để xác minh feature trên ứng dụng chạy thật.

2. Chi tiết hai phase

Mỗi phase có mục tiêu riêng nhưng dùng chung cùng một dòng dữ liệu: yêu cầu → thiết kế → triển khai → kiểm thử → bằng chứng.

DevKit

Phân tích & lập kế hoạch

Chuẩn hóa yêu cầu, lập plan, review plan và bẻ nhỏ thành tasks có thể triển khai.

Input

  • User story
  • Feature scope
  • Constitution

Output

  • plan.md
  • data-model.md
  • tasks.md
DevKit

Triển khai & quality gates

Triển khai code, kiểm tra build, review logic, kiến trúc và bảo mật trước khi hoàn tất phase phát triển.

Input

  • tasks.md
  • contracts
  • testcases.md

Output

  • src/backend
  • src/frontend
  • final-report.md
QCKit

Automation & triage

Crawl ứng dụng, sinh Page Object, chạy Playwright spec và phân loại kết quả kiểm thử.

Input

  • APP_URL
  • Feature artifacts
  • Session test

Output

  • Locator.ts / Page.ts
  • trace, screenshot
  • triage report

4. Artifacts bàn giao

Các artifacts dưới đây là cầu nối thực tế giữa hai bộ công cụ, giúp QCKit không phải suy đoán lại phạm vi kiểm thử.

DevKit

plan.md

Mô tả hướng tiếp cận, quyết định kỹ thuật và các ràng buộc chính của feature.

DevKit

tasks.md

Danh sách công việc đã triển khai, là nguồn đối chiếu phạm vi khi kiểm thử.

Shared

testcases.md

Đầu vào quan trọng để QCKit sinh automation case và phân nhóm ưu tiên.

Shared

final-report.md

Tổng hợp kết quả phát triển, review và những điểm QC cần lưu ý khi xác minh.
